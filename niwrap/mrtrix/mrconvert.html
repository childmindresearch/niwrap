1088def mrconvert_execute(
1089    params: MrconvertParameters,
1090    execution: Execution,
1091) -> MrconvertOutputs:
1092    """
1093    Perform conversion between different file types and optionally extract a subset
1094    of the input image.
1095    
1096    If used correctly, this program can be a very useful workhorse. In addition
1097    to converting images between different formats, it can be used to extract
1098    specific studies from a data set, extract a specific region of interest, or
1099    flip the images. Some of the possible operations are described in more
1100    detail below.
1101    
1102    Note that for both the -coord and -axes options, indexing starts from 0
1103    rather than 1. E.g. -coord 3 <#> selects volumes (the fourth dimension) from
1104    the series; -axes 0,1,2 includes only the three spatial axes in the output
1105    image.
1106    
1107    Additionally, for the second input to the -coord option and the -axes
1108    option, you can use any valid number sequence in the selection, as well as
1109    the 'end' keyword (see the main documentation for details); this can be
1110    particularly useful to select multiple coordinates.
1111    
1112    The -vox option is used to change the size of the voxels in the output image
1113    as reported in the image header; note however that this does not re-sample
1114    the image based on a new voxel size (that is done using the mrgrid command).
1115    
1116    By default, the intensity scaling parameters in the input image header are
1117    passed through to the output image header when writing to an integer image,
1118    and reset to 0,1 (i.e. no scaling) for floating-point and binary images.
1119    Note that the -scaling option will therefore have no effect for
1120    floating-point or binary output images.
1121    
1122    The -axes option specifies which axes from the input image will be used to
1123    form the output image. This allows the permutation, omission, or addition of
1124    axes into the output image. The axes should be supplied as a comma-separated
1125    list of axis indices. If an axis from the input image is to be omitted from
1126    the output image, it must either already have a size of 1, or a single
1127    coordinate along that axis must be selected by the user by using the -coord
1128    option. Examples are provided further below.
1129    
1130    The -bvalue_scaling option controls an aspect of the import of diffusion
1131    gradient tables. When the input diffusion-weighting direction vectors have
1132    norms that differ substantially from unity, the b-values will be scaled by
1133    the square of their corresponding vector norm (this is how multi-shell
1134    acquisitions are frequently achieved on scanner platforms). However in some
1135    rare instances, the b-values may be correct, despite the vectors not being
1136    of unit norm (or conversely, the b-values may need to be rescaled even
1137    though the vectors are close to unit norm). This option allows the user to
1138    control this operation and override MRrtix3's automatic detection.
1139    
1140    References:
1141    
1142    .
1143    
1144    Author: MRTrix3 Developers
1145    
1146    URL: https://www.mrtrix.org/
1147    
1148    Args:
1149        params: The parameters.
1150        execution: The execution object.
1151    Returns:
1152        NamedTuple of outputs (described in `MrconvertOutputs`).
1153    """
1154    cargs = mrconvert_cargs(params, execution)

The -axes option specifies which axes from the input image will be used to form the output image. This allows the permutation, omission, or addition of axes into the output image. The axes should be supplied as a comma-separated list of axis indices. If an axis from the input image is to be omitted from the output image, it must either already have a size of 1, or a single coordinate along that axis must be selected by the user by using the -coord option. Examples are provided further below.

The -bvalue_scaling option controls an aspect of the import of diffusion gradient tables. When the input diffusion-weighting direction vectors have norms that differ substantially from unity, the b-values will be scaled by the square of their corresponding vector norm (this is how multi-shell acquisitions are frequently achieved on scanner platforms). However in some rare instances, the b-values may be correct, despite the vectors not being of unit norm (or conversely, the b-values may need to be rescaled even though the vectors are close to unit norm). This option allows the user to control this operation and override MRrtix3's automatic detection.
